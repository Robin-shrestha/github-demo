Semantic Tags

Common Semantic Tags

Semantic tags carry meaning. A <div> says nothing — a <article> tells everyone "this is self-contained content."

Tag Use it when… Don't confuse it with
<header> Introductory content for a page or section — logo, nav, headline <head> (that's metadata, not visible content)
<nav> A set of navigation links — main menu, breadcrumbs, pagination Any group of links; only use for primary navigation
<main> The dominant content of the page — use only once per page <body> — body wraps everything; main wraps the core content
<section> A thematic grouping with its own heading (e.g. "About", "Features") <div> — use div for styling hooks, section for meaning
<article> Self-contained content that makes sense on its own — blog post, card, comment <section> — article stands alone; section is part of a whole
<aside> Content tangentially related to the main content — sidebar, pull quote, ad Anything central to the page; aside is supplementary
<footer> Closing content — copyright, links, contact info (page or section level) Only the page bottom; a card can also have a <footer>
<h1>–<h6> Headings in descending importance — one <h1> per page Bold text; never skip levels (h1 → h3) for visual sizing
<p> A paragraph of text <br> — use p for blocks of text, not line breaks
<ul> / <ol> / <li> Unordered list (bullets) or ordered list (numbers) with list items Manually numbered paragraphs — always use the proper list element
<a> A hyperlink — navigation, external links, anchor jumps <button> — use a for navigation, button for actions
<button> A clickable action — submit, toggle, open modal <a> — buttons do things, links go places
<img> An image that is part of the content — always include alt Decorative images; those belong in CSS as background-image
<form> A collection of inputs the user submits — login, search, signup Wrapping every input; only wrap inputs that are submitted together
<input> / <label> A form field and its label — always pair them for accessibility Using placeholder text as the label — placeholder disappears on type
<strong> / <em> strong = important (bold), em = emphasis (italic) <b> / <i> — those are purely visual with no meaning
